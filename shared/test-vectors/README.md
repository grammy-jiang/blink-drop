# Blink-Drop shared test vectors

The executable cross-language contract between `web/` and `ios/`
(docs/01-protocol.md §10). **Both** implementations must pass these; they are the
only thing binding the two independent codebases besides the protocol document.

Generated by `web/scripts/gen-vectors.ts` (`npm run gen:vectors` in `web/`).
Regenerate only deliberately — a change here is a protocol-level change.

## Two tiers

Because gzip output is not byte-identical across implementations, vectors are
split so determinism is only required where it actually holds.

### `framing/` — deterministic, byte-exact

Each vector stores a **canonical message** (`message.cbor.hex`) — the exact
`[header, payload]` dCBOR bytes, with the payload's gzip captured, not
regenerated. Given that fixed message and `maxFragmentLength` (in `params.json`),
the UR/MUR systematic parts are fully reproducible.

- `message.cbor.hex` — the canonical message, hex.
- `parts.txt` — the expected `UR:BLINK-DROP/...` systematic parts, one per line
  (uppercased for QR alphanumeric mode).
- `params.json` — `{ urType, maxFragmentLength, seqLen, name, mediaType }`.

**Contract:** feed `message.cbor.hex` + `maxFragmentLength` through the UR
encoder → the emitted parts MUST equal `parts.txt` exactly.

### `roundtrip/` — end-to-end, digest-exact

Each vector stores an original input and its SHA-256. Each side compresses with
its own gzip (bytes may differ) and must recover the original.

- `input.bin` — the original file bytes.
- `meta.json` — `{ name, mediaType, origSize, sha256 }` (sha256 of the original, hex).

**Contract:** encode `input.bin` → decode → the recovered bytes' SHA-256 MUST
equal `meta.sha256`, and `origSize` MUST match. gzip output is treated as opaque.

## Coverage

| Vector | Tier | Exercises |
|--------|------|-----------|
| `vec-01-hello` | framing + roundtrip | small text, single fragment (`seqLen = 1`) |
| `vec-02-multi` | framing | multi-fragment message (real fountain partitioning) |
| `vec-02-binary` | roundtrip | non-UTF-8 binary bytes |
| `vec-03-incompressible` | roundtrip | tiny incompressible input (`compression = 0` store path) |
